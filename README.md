# MGS V2-4.3：M4 工程方案生成与推荐逻辑精修版

## 这一版完成了什么
V2-4.3 在 V2-4.2 的基础上，对 M4 工程方案定型器的核心逻辑做了三处精修：

### 1. 统一 M4 残余风险诊断字段口径
- 将 `diagnoseResidualRisk()` 中的风险字段从简单 boolean 升级为 `{ active, level }` 结构
- 每个风险维度（功率/能量/服务/储能）各自计算 low / medium / high 等级
- 前端 `renderM4Summary()` 统一展示各风险维度的激活状态与等级
- 修正前后端风险展示映射，确保页面显示与方案生成依据一致

### 2. S4 综合平衡方案升级为风险等级自适应
- S4 不再使用固定中档系数（0.6~0.75 倍）
- 改为按每类风险的等级（low/medium/high）调用对应的增量规则：
  - low → 轻量补量（基准的 ~40%）
  - medium → 中档补量（基准的 ~65%）
  - high → 大幅补量（基准的 ~90%）
- S4 真正变成”按实际残余风险强度组合生成”的综合平衡方案

### 3. 推荐逻辑调整为”硬可行性优先”
- 新增 `isM4ScenarioFeasible()`：检查越限 ≤ 0、缺口 ≤ 1 kWh、SC ≥ 8%、服务率 ≥ 99%
- 推荐分两阶段：
  - 先筛硬可行方案，取其中综合评分最高者
  - 若全不合格，取相对最优作为 fallback，并明确标注
- 页面推荐区域同步展示硬可行方案数量和 fallback 状态
- 修复”低投资但有缺口”的方案被错误推荐的问题

当前版本已经接入可运行的工程近似指标：
- PVUR proxy：基于全年光伏发电与估算弃光量构造
- GFF proxy：基于峰/平/谷购电结构构造

后续若要严格对齐文献公式，可以再精修这两个指标口径。

## V2-4.3 M4 工程方案生成与推荐逻辑精修
1. 统一 M4 残余风险诊断字段口径，修正前后端风险展示映射；
2. 将 S4 综合平衡方案由固定中档组合升级为按风险等级自适应生成；
3. 将最终推荐逻辑调整为”硬约束可行性优先、综合评分其次”，提升工程方案定型可靠性。


## V2-4.4 M4 风险分型与方案解释增强
1. 修正 `powerLevel` 等级误判：新增 `maxRiskLevel()` 聚合多维度风险等级，避免低值维度覆盖高值维度；
2. 拆分 `serviceRisk`：`serviceRisk` 专指接入型服务风险（排队拥堵），`deliveryServiceRisk` 代表供电不足导致的服务风险；
3. S4 纳入 `storageRisk`：SOC 安全边界风险也触发储能加固，不会遗漏储能韧性问题；
4. 硬可行性加入全年 SOC 检查：`isM4ScenarioFeasible()` 增加 `monthsWithSocRisk <= 0`；
5. S3 触发逻辑精修：只在真实接入拥堵时加桩/加矩阵，不再被 `residualUnmet` 误导；
6. 方案解释增强：S1~S4 的 `intent` 根据风险状态动态生成，新增 `triggerBasis` 字段记录每个方案的触发依据。


## V2-4.5 全年无断点连续仿真内核升级
1. 新增全年工具常量：`MONTH_DAYS` / `TICKS_PER_DAY` / `DAYS_PER_YEAR` / `TICKS_PER_YEAR` / `getMonthIndexByDay()`；
2. 新增 `generateAlignedAnnualLedger()`：全年 365 天一次性生成充电事件流，固定车队 SOC 跨天继承，接入月份占用率；
3. 新增 `runFlexibleMatrixAnnualDispatch()`：全年连续柔性调度（35040 tick），储能 SOC / 车队状态跨月连续；
4. 新增 `runTraditionalPileAnnualDispatch()`：全年连续传统桩站调度，结构与月度版一致；
5. 重写 `runAnnualValidation()`：不再循环 12 次月仿真，直接调用全年连续调度函数，月度统计从全年运行中切片汇总。
- 全年仿真保证 TMY 气象 / 储能 SOC / 固定车队 SOC 三者全年连续不间断；
- 月度调度函数（`runFlexibleMatrixDispatch` / `runTraditionalPileDispatch`）保留用于 M4 压力月复验。
